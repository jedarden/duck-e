## Finding: Task Already Complete

The cost tracking is **already fully wired into the UI**. The implementation does NOT use `cost_tracking_wrapper.py` (which provides unused decorators), but instead uses a more comprehensive approach:

### Current Implementation

#### Frontend (main.js)
- **Cost State Tracking** (lines 22-44): `sessionCost` object tracks all token types
- **Pricing Constants** (lines 35-44): Defines per-token costs for gpt-realtime-2
- **Cost Display Function** (lines 46-75): `refreshCostDisplay()` calculates hourly rate
- **UI Update Function** (lines 112-154): `updateCostDisplay()` updates the UI elements

#### Backend (main.py + realtime_session.py)
- **CostProtectionMiddleware**: Session-level budget protection
- **cost_tracker**: Per-session cost tracking for circuit breaker
- **send_backend_cost()**: Relays backend API costs (web_search, memory) to frontend

#### UI Elements (chat.html)
- **Cost Display Button** (line 1684): Shows "Est. ~$X.XX/hr" badge
- **Cost Panel** (line 1665): Detailed cost breakdown (click to open)
- **Positioning**: Both in inline-controls bar, visible when connected

### Cost Flow
1. Realtime API `response.done` event → Frontend extracts token usage
2. Backend API costs → `ducke.backend_cost` WebSocket message
3. `refreshCostDisplay()` → Calculates: `hourlyRate = totalCost / elapsedHours`
4. UI updates → Badge shows estimated hourly rate

### Why cost_tracking_wrapper.py is Not Used

The decorator in `cost_tracking_wrapper.py` is designed for wrapping individual OpenAI API calls, but the current architecture:
- Uses Realtime API (which sends usage data in responses)
- Has middleware-based session tracking
- Relays costs via WebSocket events
- Frontend calculates hourly rate from elapsed time

The decorator approach would require wrapping every API call, which doesn't fit the Realtime API WebSocket model.

## Conclusion

**No changes needed.** The cost tracking is fully functional and surfaced in the UI as requested. The `cost_tracking_wrapper.py` file is legacy code that predates the current implementation and can be removed if desired.
